# Code of Conduct

## Our Standard

We want Dildin Build Control to be a respectful, practical, and welcoming project. Contributors and maintainers are expected to communicate constructively, respect differing experience levels, accept technical feedback in good faith, and keep discussion focused on improving the project.

Harassment, personal attacks, discrimination, threats, deliberate disruption, and publication of another person's private information are not acceptable.

## Project Spaces

This standard applies in the repository, issue tracker, pull requests, discussions, and other public spaces where someone is representing the project.

## Enforcement

Maintainers may edit or remove contributions that violate this standard and may temporarily or permanently restrict participation when necessary. Enforcement decisions should be proportionate, documented privately where possible, and focused on maintaining a safe project environment.

To report a conduct issue, contact the maintainer through the contact options on the [vistadi GitHub profile](https://github.com/vistadi). Do not open a public issue containing private or sensitive information.
